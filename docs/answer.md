> 実行対象ファイルは常に /<contestId>/<taskId>/main.py 固定で良いですか？既存 main.py がある場合のテンプレートコピーは上書き/スキップ/確認のどれにしますか？

ファイルのディレクトリはそれで構いません。既存ファイルがある場合はスキップしてください。

> CPython/PyPy は PATH の python/pypy3 を既定として良いですか？仮想環境が有効な場合は venv を優先して良いですか？設定キー名の希望はありますか？

はい。

> 出力比較の既定はどれにしますか？ exact（完全一致）/ trim（末尾空白・末尾改行差分無視）/ tokens（空白区切り同値）

exact にしましょう。

> タイムアウトは timeLimit(ms) \* 1.2 を既定にしますか？倍率や固定上限の指定はありますか？

既定値はそのとおりで構いません。

> 実行のカレントディレクトリは /<contestId>/<taskId> で問題ないですか？ワークスペースルートでの実行を選べる必要はありますか？

実行のカレントディレクトリはワークスペースのルートです。

> interactive: true の問題は UI 上で「非対応」として実行対象外で良いですか？

ひとまず非対応で構いません。

> Sidebar のレイアウト（配色、折りたたみ既定、結果の並び順など）に希望はありますか？なければ提案仕様通りに実装します。

特にないです。

> Console は stderr のみ表示で良いですか？stdout も併記しますか？長い出力の省略ポリシーは必要ですか？

stderr のみで構いません。stdout は Actual に記載してください。スタックトレースの省略はしないでください。
ただし、「Warning: cannot find your CPU L2 & L3 cache size in /sys/devices/system/cpu/cpuX/cach」と言った警告がほぼ確実に起きるので、これは省略してください。

> 既存の n.in/out がある場合、新規取得時は上書き・スキップ・マージ（新規は末尾採番）のどれにしますか？

マージしてください。

> 全テスト実行は直列で良いですか？並列実行の要望はありますか？

まずは直列で作成して、コミットしてください。その後から並列化を試みてください。

> 対応 OS は Windows/macOS/Linux 全てですか？既定コマンド名（python/python3）の扱いはどうしますか？

とりあえずは devcontainer 上の ubuntu で動けばよいです。CPython は python コマンド、PyPy は pypy3 コマンドです。

> 実行結果の永続化（直近の結果を JSON へ保存等）は必要ですか？不要ならメモリのみで運用します。

メモリのみでの運用で OK です。

> コマンド名や設定キーに命名の希望があれば教えてください。

特にありません。

> 文字コードは UTF-8 前提で問題ないですか？

はい。
